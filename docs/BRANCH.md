# ブランチ戦略

## 基本

[GitFlow](https://qiita.com/KosukeSone/items/514dd24828b485c69a05)に準ずる。

ハッカソンでスピード重視のため、今回は dev 環境と stg 環境と prod 環境を分けない。

## ブランチ一覧

| ブランチ名      | 解説                                                                                                    |
| --------------- | ------------------------------------------------------------------------------------------------------- |
| main            | 直接触らない。                                                                                          |
| develop         | default ブランチ。main から切る。                                                                       |
| feature/\*\*    | 実装を行うブランチ。develop から切る。                                                                  |
| release/v*.*.\* | リリースを行う際に使うブランチ。develop から切る。                                                      |
| hotfix/\*\*     | リリース時のバグ修正だけで使用するブランチ。それ以外のバグ修正は全て feature で行う。develop から切る。 |
